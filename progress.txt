✅ 1. What is JavaScript?
JavaScript is a language that makes websites work.
👉 HTML = structure
👉 CSS = design
👉 JavaScript = actions (dynamic behaviour)
Example:
Click a button → something happens
Login → check username/password
Add to cart
All this is done using JavaScript.

✅ 2. Variables (very simple explanation)
A variable is like a container where you store your data.
Example:
Imagine a container named age where you store your age:
let age = 25;

3 types of variables:
✔ let (most used)
You can change its value later.
let city = "Agra";
city = "Delhi";  // allowed

✔ const (value cannot change)
const country = "India";
country = "India"; // okay
// country = "USA";  // ❌ NOT allowed

✔ var (old method, we don’t use it today)
var name = "Kaushal";

👉 For now, always use let and const only.

✅ 3. Data Types (very simple)
✔ Number
let age = 24;
let price = 199.99;

✔ String (text inside quotes)
let name = "Rahul";
let city = "Agra";

✔ Boolean (true or false)
let isMarried = false;
let isDeveloper = true;

✔ Undefined
Variable created but no value given.
let x;
console.log(x);  // undefined

✔ Null
Empty value (we set it manually)
let data = null;

*undefined = value not assigned
*null = value assigned as empty intentionally


✅ 4. Basic Maths in JavaScript
let a = 10;
let b = 3;

console.log(a + b);  // 13
console.log(a - b);  // 7
console.log(a * b);  // 30
console.log(a / b);  // 3.33
console.log(a % b);  // 1   (remainder)
console.log(a ** b); // 1000 (power)


✅ 5. Functions (very simple explanation)
A function is like a machine.
You give input → It gives output.
Example:
A function to add two numbers:
function add(a, b) {
    return a + b;
}

console.log(add(5, 3)); // 8

Example with text:
function greet(name) {
    return "Hello " + name;
}

console.log(greet("Kaushal")); // Hello Kaushal

✅ 1. What is an Array? (Very Simple)
An array is a list where you can store multiple values in one variable.

console.log(fruits[0]); // Apple
console.log(fruits[1]); // Banana
console.log(fruits[2]); // Mango

📌 Formula:
arrayName[index]


✅ 5. Update Array Value

// ["Apple", "Orange", "Mango"]

⭐ IMPORTANT ARRAY METHODS

✅ 7. push() — Add item at END
let fruits = ["Apple", "Banana"];

fruits.push("Mango");

console.log(fruits);
// ["Apple", "Banana", "Mango"]


✅ 8. pop() — Remove item from END
let fruits = ["Apple", "Banana", "Mango"];

fruits.pop();

console.log(fruits);
// ["Apple", "Banana"]


✅ 9. map() — Transform each item (IMPORTANT)
👉 map() creates a new array
Example: multiply each number by 2
let numbers = [1, 2, 3];

let result = numbers.map(function(num) {
    return num * 2;
});

console.log(result);
// [2, 4, 6]

🧠 Think:
"Do something to each item"

✅ 10. filter() — Keep only matching items
👉 filter() removes unwanted data
Example: keep numbers greater than 18
let ages = [10, 20, 25, 15];

let adults = ages.filter(function(age) {
    return age >= 18;
});

console.log(adults);
// [20, 25]

🧠 Think:
"Filter means select"

🔁 map vs filter (Very Important)
Method
Purpose
map
Change data
filter
Select data


⭐ Arrow Function (Simple Way)
Same code shorter:

✅ reduce() — Combine all values into ONE (Simple)
🧠 Think:
reduce = “reduce many values into one value”
Example: Add all numbers

👉 Step by step:

0 + 1 = 1
1 + 2 = 3
3 + 3 = 6

🧠 Before We Start — One Important Concept
❓ Why do we need Promises / Async?
JavaScript is fast, but some things take time:
Calling an API
Getting data from server
Reading file
Example:
You order food 🍔 → you don’t get it immediately
So JavaScript says:
👉 “I’ll continue other work, and when food arrives, I’ll handle it”
This is called asynchronous programming.

✅ 1. PROMISES (VERY SIMPLE)
🔹 What is a Promise?
A Promise means:
“I promise I’ll give you the result later”
Promise has 3 states:
Pending
Fulfilled (success)
Rejected (error)

✅ Simple Promise Example
let myPromise = new Promise((resolve, reject) => {
  let success = true;

  if (success) {
    resolve("Data received");
  } else {
    reject("Error occurred");
  }
});

myPromise
  .then(result => {
    console.log(result);
  })
  .catch(error => {
    console.log(error);
  });

🧠 Explanation:
resolve → success
reject → failure
.then() → handle success
.catch() → handle error

✅ Real-Life Example (Order Food)
let orderFood = new Promise((resolve, reject) => {
  let restaurantOpen = true;

  if (restaurantOpen) {
    resolve("Food delivered 🍕");
  } else {
    reject("Restaurant closed ❌");
  }
});

orderFood
  .then(msg => console.log(msg))
  .catch(err => console.log(err));


⭐ 2. ASYNC / AWAIT (EASIEST WAY)
🧠 Async/Await = Cleaner way to use Promises

❌ Promise style (hard for beginners)
fetch(url)
  .then(res => res.json())
  .then(data => console.log(data));


✅ Async/Await style (easy)
async function getData() {
  let response = await fetch(url);
  let data = await response.json();
  console.log(data);
}

🧠 Think:
async → this function uses async code
await → wait here until result comes

✅ Simple Example
async function greet() {
  return "Hello World";
}

greet().then(msg => console.log(msg));

✅ 1️⃣ What is TypeScript? (Very Simple)
👉 TypeScript = JavaScript + rules
JavaScript allows anything:
let age = 25;

TypeScript does NOT allow this.
let age: number = 25;
age = "twenty five"; // ❌ Error

🧠 Think:
TypeScript protects you from mistakes before running code

✅ 2️⃣ Why TypeScript is used
Avoid bugs
Better code quality
Required for Angular

🔹 Array

🔹 Any (Avoid if possible)

🔹 Union (One of many types)
let id: number | string;
id = 101;
id = "A101";

⭐ DAY 5 — TYPESCRIPT INTERVIEW PRACTICE (Q → A)

Q1️⃣ What is TypeScript?
✅ Answer:
TypeScript is a superset of JavaScript that adds static typing.
It helps catch errors at compile time and makes code more maintainable and scalable.

Q2️⃣ Why do we use TypeScript instead of JavaScript?
✅ Answer:
We use TypeScript to:
Avoid runtime errors
Write type-safe code
Improve readability
Work efficiently in large projects like Angular

Q3️⃣ What are basic data types in TypeScript?
✅ Answer:
Some basic TypeScript data types are:
number
string
boolean
array
any
union
enum

Q4️⃣ How do you declare a variable with type in TypeScript?
✅ Answer:
let age: number = 25;
let name: string = "Rahul";


Q5️⃣ What is any type?
✅ Answer:
any allows a variable to store any type of value.
It disables type checking and should be avoided if possible.
let data: any = 10;
data = "hello";


Q6️⃣ What is a union type?
✅ Answer:
Union type allows a variable to have more than one type.
let id: number | string;
id = 101;
id = "A101";


Q7️⃣ What is an interface?
✅ Answer:
An interface defines the structure of an object.
It ensures that an object has required properties with correct types.

Q8️⃣ Create an interface User and object using it.
✅ Answer:
interface User {
  id: number;
  name: string;
  email: string;
}

let user1: User = {
  id: 1,
  name: "Kaushal",
  email: "test@gmail.com"
};


Q9️⃣ What is an optional property in interface?
✅ Answer:
Optional property is not mandatory and is marked using ?.
interface Employee {
  name: string;
  salary?: number;
}

import { HttpClient } from '@angular/common/http';
import { Injectable } from '@angular/core';

@Injectable({
 providedIn: 'root'
})
export class UserService {

 apiUrl = 'http://localhost:3000/users';

 constructor(private http: HttpClient) {}

 getUsers() {
   return this.http.get(this.apiUrl);
 }

 addUser(data: any) {
   return this.http.post(this.apiUrl, data);
 }
}

⚙️ STEP 4: Use in Component
📄 app.component.ts

@Component({

   <ul>

     </li>

 `
})
export class AppComponent {

 users: any[] = [];

 constructor(private userService: UserService) {}

 loadUsers() {
   this.userService.getUsers().subscribe((res: any) => {
     this.users = res.data;
   });
 }
